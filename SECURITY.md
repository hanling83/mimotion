## 漏洞报告

如果你发现了安全漏洞，请通过以下方式报告：

1. 在本仓库中提交 [Issue](../../issues)，标题以 `[安全]` 开头
2. 或者通过邮件联系维护者

### 报告内容建议

- 漏洞描述
- 复现步骤
- 影响范围
- 修复建议（如有）

### 响应流程

- 收到报告后 **7 个工作日内** 回复确认
- 确认漏洞后尽快修复并发布更新
- 修复完成后在 Issue 中公开致谢（除非报告者要求匿名）

## 安全建议

- 务必保管好你的 `AES_KEY` 和 `CONFIG` 密钥
- 不要将密钥提交到公开仓库
- 定期更换 GitHub Personal Access Token
- 定期更换 Zepp Life 密码
